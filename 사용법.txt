============================================================
  VibeGraph - 바이브코딩 대화 순도 분석기
============================================================
  ▶ 보기 편한 HTML 가이드: 가이드.html 을 더블클릭하세요.
    (명령어 복사 버튼 포함)
  ▶ 별도 API 키·추가 결제 없음. 채점은 Claude Code 창에서!
------------------------------------------------------------

[ 처음 설치하는 사람 (다른 PC) ]
------------------------------------------------------------
1. 이 'vibegraph' 폴더를 통째로 원하는 위치에 복사합니다.
   (C드라이브가 아니어도 됩니다. 예: D:\tools\vibegraph)
   * 단, 한글/공백 없는 경로를 권장합니다.

2. 폴더 안의  install.bat  을 더블클릭합니다.
   - Python 확인 → PATH 등록 까지 자동 진행됩니다.

3. Python이 없다는 메시지가 뜨면:
   https://www.python.org/downloads/ 에서 설치
   (설치 첫 화면 "Add Python to PATH" 체크 필수!)
   → 설치 후 install.bat 다시 실행

4. 설치가 끝나면 *새 터미널(PowerShell/CMD)* 을 엽니다.

[ "vibe 명령을 못 찾는다"면 ]
------------------------------------------------------------
  설치는 됐는데 'vibe : 용어가 인식되지 않습니다' 가 뜨면,
  install.bat을 다시 할 필요 없습니다. 그 터미널이 옛 환경변수를
  들고 있는 것뿐입니다.
   - 특히 VS Code는 켜질 때 환경변수를 한 번 읽어두므로, PATH 등록
     이후 터미널 탭을 새로 열어도 계속 못 찾을 수 있습니다.
   - 해결: VS Code를 완전히 종료 후 재실행하거나, 외부 PowerShell
     창을 새로 열어 사용하세요. 그 뒤로는 계속 동작합니다.
   - 재부팅 후에도 안 되면 VS Code가 이전 세션을 복원하며 옛 환경을
     이어받은 경우입니다. 한 번 완전히 껐다 켜면 해결됩니다.


[ 사용법 ]
------------------------------------------------------------
  ※ 어디에 입력? : vibe 로 시작하는 명령은 전부 '터미널'에서.
    클로드창에 들어가는 건 vibe prompt가 복사해준 '프롬프트'뿐.
    (Claude를 터미널에서 claude 로 띄웠다면, 터미널을 하나 더
     열어 거기서 vibe 명령을 치세요. Claude 창은 그대로 둔 채.)

  vibe start  <프로젝트명>  <작업명>
      예) vibe start eGov 로그인_세션만료수정
      → 작업 폴더가 생성됩니다.

  (Claude Code로 평소처럼 작업)

  작업이 끝나면 *같은 창에서*:
  vibe prompt
      → 채점 프롬프트가 클립보드에 복사되고 result.json 이 열립니다.
      → 작업하던 그 Claude 창에 붙여넣기(Ctrl+V) 하면
         Claude가 이번 대화를 스스로 채점한 JSON을 출력합니다.
      ※ 대화를 직접 긁어 복사할 필요 없습니다. Claude가 이미
         맥락을 갖고 있으니 '같은 창'에서 채점하는 게 핵심입니다.

  → Claude가 출력한 JSON을 result.json 에 붙여넣고 저장

  vibe end
      → result.json 으로 report.html 을 만듭니다.
      → report.html 더블클릭하면 브라우저로 열립니다.

  vibe list              전체 작업 목록 보기
  vibe stats             최근 3주 통계
  vibe stats --weeks 8   최근 8주 통계
  vibe dashboard         전체 작업 조회 페이지 생성/열기 (= view, index)
  vibe growth            누적 성장 리포트(추세·최약점·반복스멜) 생성/열기
  vibe coach             누적 신호 기반 코칭 프롬프트 복사 (Claude창에서 코칭)


[ 전체 작업 한눈에 보기 ]
------------------------------------------------------------
  채점한 모든 작업을 표 하나로 모아 보는 조회 페이지가 있습니다.

    vibe dashboard
      → vibegraph\index.html 이 만들어지고 브라우저로 열립니다.
      → vibe view / vibe index 로 쳐도 동일하게 동작합니다.

  조회 페이지 기능:
    - 상단 요약: 총 작업 수 / 전체 평균 점수 / 프로젝트 수
    - 검색창: 프로젝트·작업명 실시간 필터
    - 프로젝트 드롭다운으로 좁혀 보기
    - 열 제목 클릭으로 날짜·점수·턴·토큰 정렬
    - 등급 배지(S·A·B·C), 턴 수·추정 토큰 표시
    - 작업명 클릭 → 그 작업의 개별 리포트가 새 탭으로 열림

  ※ vibe end 할 때마다 index.html 이 자동 갱신됩니다.
    vibegraph\index.html 을 즐겨찾기 해두고 더블클릭만 해도
    늘 최신 목록이 보입니다.


[ 성장 추적 & 코칭 ]
------------------------------------------------------------
  작업이 쌓이면 여러 리포트를 누적 분석할 수 있습니다.
  두 가지로 나뉩니다.

  (1) 성장 리포트 — 계산 기반, 환각 없음
        vibe growth          (기본: 최근 2주만 집계)
      → growth.html 생성/열기
      → 점수 추세 그래프, 항목별 평균과 '가장 약한 항목',
        반복된 Prompt Smell, 프로젝트별 평균, 등급 분포.
      → AI 해석이 아닌 순수 계산이라 정직합니다.

      ※ 기본 기간이 '최근 2주'인 이유:
        전체를 기본으로 하면 작업이 쌓일수록 느려지고, 추세 차트의
        점이 너무 많아져 보기 어려워집니다. 그래서 vibe end 가 자동
        생성하는 페이지도 최근 2주만 집계합니다.

      [조건별로 보기]
        vibe growth                          최근 2주(기본)
        vibe growth --all                    전체 기간
        vibe growth --weeks 8                최근 8주만
        vibe growth --project <프로젝트명>     특정 프로젝트(기간 함께 적용)
        vibe growth --project eGov --all     특정 프로젝트 전체 기간
      ※ 직접 타이핑할 필요 없이, growth.html 하단의
        '다른 조건으로 보기'에서 명령을 복사해 붙여넣으면 됩니다.
        (실제 프로젝트 목록 기준으로 자동 생성)

  (2) 코칭 — 맞춤 조언 (Claude창 사용)
        vibe coach
      → 계산해 둔 신호(최약점·반복스멜·추세 등)를 담은
        코칭 프롬프트를 클립보드에 복사합니다.
      → Claude 창에 붙여넣으면 그 수치에 근거한
        '반복 약점 + 다음 작업 행동 과제'를 받습니다. (API 불필요)
      ※ 맨 숫자를 던져 통찰을 기대하는 게 아니라, 우리가 신호를
        먼저 뽑고 AI는 코칭 문장으로만 풀어주는 방식 → 억지분석 방지.
      ※ 데이터가 적으면(4개 미만) 코치가 '단정 어렵다'고 답할 수 있음.

  ※ vibe end 할 때마다 growth.html 도 함께 자동 갱신됩니다.
    (자동 생성은 '최근 2주' 기준 — 속도·가독성. 전체는 vibe growth --all)
    index.html 과 growth.html 은 상단 링크로 서로 오갈 수 있습니다.


[ 여러 작업을 연달아 할 때 ]
------------------------------------------------------------
  ※ 한 작업을 vibe end 까지 끝낸 뒤 다음 작업을 이어서 할 때,
    채점 정확도를 위해 신경 쓸 곳은 '터미널'이 아니라
    'Claude 대화창'입니다.
      - 터미널: 맥락(기억) 없음 → 닫을 필요 없이 그대로 vibe start.
      - Claude 대화창: 이전 작업 대화를 기억함 → 그대로 두면
        다음 작업 채점에 이전 대화가 섞일 수 있음.

  [ 케이스 A — 서로 독립적인 작업 ]
    (예: 로그인 수정 → 게시판 페이징)
    다음 작업 전에 Claude 대화창의 맥락을 비웁니다.

      1) Claude 대화창에 입력:   /clear     (또는 새 대화 열기)
      2) 터미널에서 다른 이름으로:  vibe start eGov 게시판_페이징수정

    [작업01] vibe start → 작업 → vibe prompt → 채점 → vibe end
        |
        +-- Claude 대화창에서 /clear  (터미널은 그대로 둠)
        |
    [작업02] vibe start(다른 이름) → 작업 → vibe prompt → 채점 → vibe end

  [ 케이스 B — 맥락이 이어지는 작업 ]
    (예: 회원 목록 페이지 → 회원 상세 페이지)
    앞 작업의 코드/결정을 이어가야 하면 /clear 하지 마세요.
    두 가지 방법 중 하나를 권장합니다.

    방법 1 (가장 깔끔, 추천)
      - 이어지는 작업을 '하나의 세션으로 묶어' 한 번만 채점.
      - 작업명을 포괄적으로 (예: 회원페이지_목록+상세) 잡고,
        전체가 끝난 뒤 vibe prompt → vibe end 를 한 번만 실행.
      - 연결된 흐름을 한 덩어리로 평가 → 점수가 가장 정확.

    방법 2 (각각 점수를 내고 싶을 때)
      - 맥락은 유지(/clear 안 함)하되, 작업02에서 vibe prompt 로
        받은 채점 프롬프트 '맨 앞에 한 줄'을 덧붙여 알려줍니다:

        ※ 직전 작업(목록 페이지)은 이미 채점했다. 그 이후의
          "회원 상세 페이지" 작업 대화만 평가 대상으로 삼아라.

  정리: 맥락 끊어도 되면 케이스 A(/clear),
        맥락 이어야 하면 케이스 B.
        강하게 연결된 작업은 방법 1(한 번에 묶어 채점)이 가장 단순.


[ 왜 API 키가 필요 없나 ]
------------------------------------------------------------
  채점을 작업하던 Claude Code 창에서 하기 때문입니다.
  (Pro/Max 구독으로 커버 → 별도 종량제 결제 불필요)
  이 CLI는 폴더정리·HTML생성·통계만 담당합니다.


[ 데이터 저장 위치 ]
------------------------------------------------------------
  채점 데이터는 코드와 분리되어 사용자 폴더에 쌓입니다.
   ~/.vibegraph\   (= %USERPROFILE%\.vibegraph)
   ├ index.html        ★ 전체 작업 조회 페이지 (vibe dashboard)
   ├ growth.html       ★ 누적 성장 리포트 (vibe growth)
   └ <프로젝트명>\
       └ <날짜_작업명>\
           ├ result.json   Claude가 출력한 채점 결과(붙여넣기)
           ├ data.json     최종 점수 데이터(통계 연동)
           └ report.html   리포트

  - 저장 위치를 바꾸려면 환경변수 VIBE_HOME 에 경로를 지정하세요.
  - 예전처럼 코드 폴더에 데이터가 있던 경우, 첫 실행 때 사용자
    폴더로 자동 이전됩니다.
