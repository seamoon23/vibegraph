# 바이브코딩, 감으로 하지 말고 점수로 보자 — VibeGraph 만든 이야기

> AI랑 같이 코딩한 대화를 매번 스스로 채점하고, 내 프롬프트 습관이 좋아지는지 그래프로 추적하는 CLI를 만들었습니다.
> **API 키도, 추가 결제도 필요 없습니다.** GitHub: <https://github.com/seamoon23/vibegraph>

## 시작은 "왜 자꾸 같은 실수를 하지?"

요즘 Claude Code 같은 도구로 'AI랑 같이' 코딩합니다. 흔히 바이브코딩이라고 하죠.
편한데, 돌아보면 매번 비슷한 데서 시간을 흘립니다.

- 요구사항을 한 번에 안 주고 → 여러 번 고치게 만듦
- 모호하게 지시 → 엉뚱한 결과 → 다시 설명
- AI가 던진 땜질·우회 제안에 그냥 끌려감
- 환경(버전·파일 구조)을 안 알려줘서 → 안 맞는 코드를 받음

문제는 **그때뿐이고 기록이 안 남는다**는 거였습니다. 다음 작업에서 또 똑같이 합니다.
"이걸 매 작업이 끝날 때마다 객관적으로 점수 매겨주면, 습관이 고쳐지지 않을까?" 그래서 만들었습니다.

## VibeGraph가 하는 일

작업이 끝나면 **그 대화를 4개 항목(각 25점)으로 채점**합니다.

- **원샷 달성률** — 요구사항·제약을 한 번에 줘서 불필요한 수정 턴을 줄였나
- **컨텍스트 유지력** — 초기 목표를 유지하고 무의미한 왕복이 없었나
- **주도권 제어력** — 땜질 제안에 휩쓸리지 않고 사람이 방향을 잡았나
- **프롬프트 선명도** — 모호함 없이 에러로그·소스 같은 명확한 컨텍스트를 줬나

그리고 점수만 주는 게 아니라:
- **무엇이 문제였는지**(Prompt Smell)와 **이렇게 썼으면 더 좋았다**(Good/Bad 예시)
- 이 작업엔 **다른 스킬·도구가 나았다**는 추천
- 작업이 쌓이면 **성장 추세·가장 약한 항목·반복되는 실수**까지

![개별 리포트](screenshots/report.png)

## 핵심: API 키가 필요 없습니다

여기가 이 도구의 포인트입니다. 보통 이런 채점을 하려면 별도 API 키를 발급받고 종량제로 돈을 냅니다.
저는 이미 Claude **Pro 구독** 중인데 또 결제하긴 싫었습니다.

그래서 발상을 바꿨습니다.
**채점을, 작업하던 바로 그 Claude 창에서 합니다.**
그 창은 이미 우리 대화 전체 맥락을 갖고 있으니까요.

```
[터미널]                         [Claude Code 창]
vibe start  ───────────────▶  평소처럼 코딩
vibe prompt ─(채점 프롬프트 복사)─▶ 붙여넣기 → Claude가 스스로 채점 JSON 출력
result.json 에 저장 ◀────────────┘
vibe end    ──▶ 리포트 HTML 생성
```

대화를 따로 복사해 붙일 필요도 없습니다. 프롬프트 한 줄 붙여넣으면 끝.
CLI는 폴더 정리·HTML 생성·통계만 담당하고, **외부 API를 일절 호출하지 않습니다.**

## 쌓이면 보이는 것들

작업 하나하나의 리포트도 좋지만, 진짜 가치는 **누적**에서 나옵니다.

**전체 조회 페이지** — 모든 작업을 한 표에. 검색·프로젝트·기간 필터, 클릭하면 개별 리포트로.

![전체 조회](screenshots/dashboard.png)

**성장 리포트** — 점수 추세 그래프, 항목별 평균과 *가장 약한 항목*, 반복되는 Prompt Smell, 프로젝트별 평균.
이건 **AI 해석이 아니라 순수 계산**이라 환각이 없습니다.

![성장 리포트](screenshots/growth.png)

**코칭** — 위에서 *계산으로 뽑아낸 신호*를 Claude에 넘겨, "반복되는 약점 + 다음 작업에서 당장 할 행동"을 받습니다.
맨 숫자를 던져 "통찰 좀 줘" 하면 공허한 일반론이 나오기 쉬워서, **신호를 먼저 추출하고 AI는 문장으로만 풀게** 한 게 포인트입니다.

## 만들면서 신경 쓴 설계 결정 몇 가지

개인 도구지만, 몇 가지는 일부러 고민했습니다.

- **채점은 보수적으로.** 처음엔 점수가 후하게 나왔습니다. "보통의 무난한 대화는 60~75점이 정상, A는 정말 잘한 경우만"이라고 기준을 빡세게 다시 잡았습니다.
- **데이터와 코드 분리.** 채점 데이터는 `~/.vibegraph`에, 코드는 저장소에. GitHub에 올려도 개인 기록이 안 섞이고, `git clean` 같은 명령에도 데이터가 안전합니다.
- **성장 리포트 기본은 최근 2주.** 전체를 기본으로 하면 작업이 쌓일수록 느려지고 차트도 복잡해집니다. 기본은 최근 2주, 전체는 `vibe growth --all`로.
- **명령어는 페이지에서 복사.** 필터 조건을 외워서 타이핑하는 대신, 리포트 하단에서 `vibe growth --project ...` 같은 명령을 복사 버튼으로 가져다 쓰게 했습니다.

## 설치 & 사용 (Windows)

1. 클론 후 `install.bat` 더블클릭 (Python 확인 + PATH 등록)
   ```bash
   git clone https://github.com/seamoon23/vibegraph.git
   ```
2. 새 터미널에서:
   ```bash
   vibe start myproject 로그인_세션수정
   # … Claude Code로 작업 …
   vibe prompt        # 채점 프롬프트 복사 → Claude 창에 붙여넣기 → JSON을 result.json 에 저장
   vibe end           # 리포트 생성
   vibe growth        # 성장 추세 보기
   ```

> Python 3.8+ 필요(설치 시 "Add Python to PATH" 체크). 설치 후 `vibe`가 안 잡히면 VS Code를 완전히 재시작하세요.
