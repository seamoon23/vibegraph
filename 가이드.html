■ 처음 설치하기
1

폴더 복사

이 vibegraph 폴더를 통째로 원하는 위치에 복사합니다. C드라이브가 아니어도 됩니다.

⚠ 한글·공백 없는 경로를 권장합니다. 예) D:\tools\vibegraph
2

설치 마법사 실행

폴더 안의 install.bat 을 더블클릭하면 아래가 자동으로 진행됩니다.

  • Python 설치 확인
  • vibe 명령어 PATH 등록
✅ 설치가 끝나면 반드시 새 터미널(PowerShell/CMD)을 열어야 적용됩니다.
💳 API 키·추가 결제는 필요 없습니다 — 채점은 Claude Code 창에서 합니다.
3

Python이 없다면

python.org/downloads 에서 설치하세요. 설치 첫 화면에서 아래 항목 체크가 필수입니다.

☑ Add Python to PATH

설치 후 install.bat 을 다시 실행하세요.

!

"vibe 명령을 못 찾는다"면

설치는 됐는데 vibe : 용어가 인식되지 않습니다 가 뜨면, install.bat을 다시 할 필요는 없습니다. 그 터미널이 옛 환경변수를 들고 있는 것뿐입니다.

특히 VS Code는 자신이 켜질 때 환경변수를 한 번 읽어두기 때문에, PATH 등록 이후 터미널 탭을 새로 열어도 계속 못 찾을 수 있습니다.

✅ 해결: VS Code를 완전히 종료했다가 다시 실행하거나, 외부 PowerShell 창(시작 메뉴 → PowerShell)을 새로 열어 사용하세요. 그 뒤로는 계속 동작합니다.
재부팅을 했는데도 안 된다면, VS Code가 이전 세션을 복원하면서 옛 환경을 그대로 이어받은 경우입니다. 한 번 완전히 껐다 켜면 해결됩니다.
■ 작업 흐름
vibe start → (Claude로 작업) → vibe prompt → JSON을 result.json에 저장 → vibe end → report.html

먼저 — 어디에 입력하나요? vibe 로 시작하는 명령은 전부 터미널에서 칩니다. 클로드창에 들어가는 건 ④에서 붙여넣는 "프롬프트" 하나뿐입니다.

🖥️ 터미널 (PowerShell)

VS Code 아래쪽 터미널.
vibe start · vibe prompt · vibe end · vibe stats 를 입력하는 곳.

💬 클로드창 (대화)

VS Code 오른쪽 Claude 패널.
평소 작업 + ③에서 복사된 프롬프트를 붙여넣고, 나온 JSON을 빼오는 곳.

⚠ Claude를 터미널에서 claude 로 띄워 쓰는 경우, 그 터미널은 "클로드창"이라 vibe를 못 칩니다. 터미널을 하나 더 열어 거기서 vibe 명령을 치세요(Claude 창은 그대로 둔 채).
💳 별도 API 키·추가 결제가 필요 없습니다. 채점은 작업하던 Claude Code 창에서 이뤄집니다(Pro 구독으로 커버).

터미널① 작업 시작 — 프로젝트명과 작업명을 붙여 실행하면 작업 폴더가 생깁니다.

vibe start eGov 로그인_세션만료수정

클로드창② 작업 진행 — Claude Code로 평소처럼 작업합니다.

터미널③ 채점 프롬프트 받기 — 작업이 끝나면 아래를 실행하세요. 채점 프롬프트가 클립보드에 복사되고 result.json 이 열립니다.

vibe prompt

클로드창④ Claude에 붙여넣기 — 방금 작업한 그 Claude 창에 붙여넣으면(Ctrl+V), Claude가 이번 대화를 스스로 채점해 JSON 결과를 출력합니다.

💡 직접 대화를 긁어 복사할 필요가 없습니다. Claude가 이미 대화 맥락을 갖고 있으니, 같은 창에서 채점하는 게 핵심입니다. (새 창에서 하면 맥락이 없어 부정확합니다)

클로드창→파일⑤ 결과 저장 — Claude가 출력한 JSON을 복사해서, 열려 있는 result.json 에 붙여넣고 저장합니다.

터미널⑥ 리포트 생성 — 아래 명령을 실행하면 점수·차트·스킬추천 리포트가 만들어집니다.

vibe end

터미널⑦ 확인 — 생성된 report.html 을 더블클릭하면 브라우저에서 열립니다.

■ 여러 작업을 연달아 할 때

한 작업을 vibe end 까지 끝낸 뒤 같은 자리에서 다음 작업을 이어서 할 때, 채점 정확도를 위해 신경 쓸 곳은 "터미널"이 아니라 "Claude 대화창"입니다.

🖥️ 터미널

맥락(기억)을 들고 있지 않습니다. 닫을 필요 없이 그대로 다음 vibe start 를 치면 됩니다.

💬 Claude 대화창

이전 작업 대화를 기억합니다. 그대로 두면 다음 작업 채점에 이전 대화가 섞일 수 있습니다.

케이스 A — 서로 독립적인 작업 (예: 로그인 수정 → 게시판 페이징)

작업끼리 관련이 없으면, 다음 작업 전에 Claude 대화창의 맥락을 초기화하세요. 그래야 작업02가 작업01 대화에 오염되지 않고 깨끗하게 채점됩니다.

클로드창대화창에 입력해 맥락을 비웁니다(또는 새 대화 열기):

/clear

터미널그다음 다른 작업명으로 시작합니다:

vibe start eGov 게시판_페이징수정
[작업01] vibe start → 작업 → vibe prompt → 채점 → vibe end │ ▼ 💬 Claude 대화창에서 /clear ← 맥락 분리 (터미널은 그대로) │ [작업02] vibe start (다른 이름) → 작업 → vibe prompt → 채점 → vibe end

케이스 B — 맥락이 이어지는 작업 (예: 회원 목록 페이지 → 회원 상세 페이지)

앞 작업의 코드·결정을 그대로 이어가야 한다면 /clear 하지 마세요. 맥락이 끊기면 작업 효율이 떨어집니다. 이때는 두 가지 방법 중 하나를 권장합니다.

  • 방법 ① (가장 깔끔, 추천) — 이어지는 작업을 하나의 세션으로 묶어 한 번만 채점합니다. 작업명을 포괄적으로 잡고(회원페이지_목록+상세), 전체가 끝난 뒤 vibe prompt → vibe end 를 한 번만 실행하세요. 연결된 흐름을 한 덩어리로 평가하므로 점수가 가장 정확합니다.
  • 방법 ② (각각 점수를 내고 싶을 때) — 맥락은 유지(/clear 안 함)하되, 작업02에서 vibe prompt 로 받은 채점 프롬프트 맨 앞에 한 줄을 덧붙여 "직전 작업 이후(작업02) 대화만 평가하라"고 Claude에 알려줍니다.

클로드창방법 ②로 채점할 때, 붙여넣은 프롬프트 맨 위에 이 문장을 먼저 적어주세요:

※ 직전 작업(목록 페이지)은 이미 채점했다. 그 이후의 "회원 상세 페이지" 작업 대화만 평가 대상으로 삼아라.
🧭 정리: 맥락 끊어도 되면 케이스 A(/clear), 맥락 이어야 하면 케이스 B. 특히 강하게 연결된 작업은 방법 ①(한 번에 묶어 채점)이 가장 단순하고 정확합니다.

■ 성장 추적 & 코칭

작업이 쌓이면 여러 리포트를 누적 분석해 성장 흐름과 약점을 볼 수 있습니다. 두 가지로 나뉩니다.

터미널① 성장 리포트 (계산 기반, 정직함)

점수 추세 그래프, 항목별 평균과 가장 약한 항목, 반복된 Prompt Smell, 프로젝트별 평균, 등급 분포를 보여줍니다. AI 해석이 아니라 순수 계산이라 환각이 없습니다.

vibe growth
⏱ 기본은 "최근 2주"만 집계합니다. 전체를 기본으로 하면 작업이 쌓일수록 느려지고 추세 차트도 복잡해지기 때문입니다. vibe end 가 자동 생성하는 페이지도 최근 2주 기준입니다.

전체 기간이나 특정 프로젝트를 보려면 조건을 붙이세요.

vibe growth --all
vibe growth --project eGov --all
💡 직접 칠 필요 없이, 생성된 growth.html 하단의 "🔎 다른 조건으로 보기"에서 전체·프로젝트·기간별 명령을 복사 버튼으로 가져다 쓰면 됩니다. (실제 프로젝트 목록 기준으로 자동 생성)

터미널 + 클로드창② 코칭 (맞춤 조언)

위에서 계산해 둔 신호(최약점·반복스멜·추세 등)를 담은 코칭 프롬프트를 클립보드에 복사합니다. 이걸 Claude 창에 붙여넣으면, 그 수치에 근거한 반복 약점 + 다음 작업 행동 과제를 받습니다.

vibe coach
💡 ②는 "맨 숫자를 AI에게 던져 통찰 기대"가 아니라, 우리가 신호를 먼저 추출하고 AI는 그걸 코칭 문장으로만 풀어주는 방식입니다. 그래서 억지 분석을 피합니다. 데이터가 적으면(4개 미만) 코치가 "단정 어렵다"고 솔직히 답할 수 있습니다.
✅ vibe end 할 때마다 growth.html 도 함께 자동 갱신됩니다. 조회 페이지(index.html)와 성장 리포트는 상단 링크로 서로 오갈 수 있습니다.
■ 전체 작업 한눈에 보기

채점한 모든 작업을 표 하나로 모아 보는 조회 페이지가 있습니다. 아래 명령을 터미널에서 실행하면 ~/.vibegraph\index.html 이 만들어지고 브라우저로 열립니다.

vibe dashboard
  • 📊 상단 요약 — 총 작업 수 · 전체 평균 점수 · 프로젝트 수
  • 🔍 검색창 — 프로젝트·작업명으로 실시간 필터
  • 🗂 프로젝트 드롭다운 으로 좁혀 보기
  • ↕ 열 제목 클릭 으로 날짜·점수·턴·토큰 정렬
  • 🏷 등급 배지(S·A·B·C)와 턴 수·추정 토큰 표시
  • 🔗 작업명을 클릭 하면 그 작업의 개별 리포트가 새 탭으로 열림
✅ vibe end 를 할 때마다 index.html 이 자동 갱신됩니다. ~/.vibegraph\index.html 을 즐겨찾기 해두고 더블클릭만 해도 늘 최신 목록이 보입니다.
💡 vibe dashboard 대신 vibe view / vibe index 라고 쳐도 똑같이 동작합니다.
■ 데이터 저장 위치

채점 데이터는 코드와 분리되어 사용자 폴더 ~/.vibegraph (= %USERPROFILE%\.vibegraph) 에 프로젝트별로 쌓입니다.

~/.vibegraph\ ├ index.html ★ 전체 작업 조회 페이지 (vibe dashboard) ├ growth.html ★ 누적 성장 리포트 (vibe growth) └ <프로젝트명>\ └ <날짜_작업명>\ ├ result.json Claude가 출력한 채점 결과(붙여넣기) ├ data.json 최종 점수 데이터(통계 연동) └ report.html 리포트
📦 저장 위치를 바꾸려면 환경변수 VIBE_HOME 에 원하는 경로를 지정하세요. 예전처럼 코드 폴더에 데이터가 있던 경우, 첫 실행 때 사용자 폴더로 자동 이전됩니다.
